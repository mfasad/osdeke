Если ваш бизнес связан с отправкой или получением грузов через СДЭК, рано или поздно возникнет вопрос автоматизации. Ручное создание заказов, отслеживание статусов и обработка документов отнимают время и увеличивают риск ошибок. Решение — интеграция СДЭК с другими сервисами, которые вы уже используете: от 1С и Bitrix24 до Ozon и Telegram.

Но какие именно интеграции принесут максимальную пользу? Где брать API-ключи, сколько это стоит и как избежатьных ошибок при настройке? В этом руководстве мы разберём все возможные варианты интеграций — от стандартных (маркетплейсы, CRM) до неочевидных (мессенджеры, системы аналитики), — а также дадим чек-листы для самостоятельной настройки. Данные актуальны на июнь 2026 года и учитывают последние обновления API СДЭК.

1. Почему интеграция с СДЭК обязательна для бизнеса

Без автоматизации работа с СДЭК превращается в рутину:

  • 📦 Вручную создавать накладные для каждого заказа на маркетплейсе.
  • 🔄 Копировать трек-номера из личного кабинета СДЭК в Excel или CRM.
  • ⏳ Отслеживать статус доставки в нескольких системах одновременно.
  • 💰 Терять деньги на ошибках в весе/габаритах из-за человеческого фактора.

Интеграция решает эти проблемы:

  • ⚡ Экономия времени: заказы создаются автоматически, трек-номера подтягиваются в CRM.
  • 📊 Прозрачность: все статусы доставки отображаются в одной системе (например, в Bitrix24 или MoySklad).
  • 💰 Экономия на тарифах: некоторые интеграции (например, с 1С) позволяют оптимизировать габариты и вес груза, снижая стоимость доставки.
  • 🛡️ Снижение ошибок: исключается ручной ввод данных — система сама подставляет адреса, вес и другие параметры.

По данным СДЭК, компании, использующие интеграции, сокращают время обработки заказов на 40–70%, а количество ошибок в накладных снижается в 3–5 раз.

2. Официальные интеграции СДЭК: что предлагает сама компания

СДЭК предоставляет несколько готовых решений для интеграции, которые не требуют глубоких технических знаний. Их можно настроить прямо из личного кабинета или через партнёров.

Тип интеграции Для кого подходит Стоимость Сложность настройки
API СДЭК (REST) Разработчики, крупный бизнес Бесплатно (лимиты по запросам) ⭐⭐⭐⭐ (требует программиста)
Модуль для 1С Оптовые и розничные компании на 1С От 5 000 ₽ (разовая оплата) ⭐⭐ (есть инструкции)

Интеграция с маркетплейсами (Ozon, WB, Яндекс Маркет) Продавцы на маркетплейсах Бесплатно (в рамках тарифа маркетплейса) ⭐⭐ (настройка в ЛК маркетплейса)

Самое простое решение — плагины для CMS. Например, для WordPress есть официальный плагин CDEK WooCommerce Shipping, который автоматически:

  • 📦 Создаёт заказы в СДЭК при оформлении покупки на сайте.
  • 🔄 Подтягивает трек-номера обратно в заказ.
  • 💰 Показывает клиенту актуальные тарифы доставки на этапе оформления.

Для 1С есть два варианта:

  1. Официальный модуль от СДЭК (платный, но с поддержкой).
  2. Самостоятельная интеграция через API (бесплатно, но требует навыков программирования).
Что делать, если нет официальной интеграции для вашей CMS?

Если ваша CMS не поддерживается (например, самописная платформа), можно использовать универсальные решения:

1. Zapier или Make (ex-Integromat) — для связки СДЭК с CRM/мессенджерами.

2. API СДЭК — для кастомной разработки.

3. Готовые коннекторы от партнёров (например, «Клеверенс» или «АпиХаб»).

3. Интеграция СДЭК с CRM: Bitrix24, amoCRM, MoySklad

CRM-системы — один из самых востребованных типов интеграций. Они позволяют:

  • 📞 Видеть статус доставки прямо в карточке клиента.
  • 📊 Автоматически создавать задачи для менеджеров (например, «Связаться с клиентом, если заказ застрял на складе»).
  • 💰 Анализировать логистические затраты в разрезе клиентов или товаров.

Рассмотрим три популярные CRM и варианты их интеграции с СДЭК:

Bitrix24

Для Bitrix24 есть два способа интеграции:

  1. Официальный коннектор от СДЭК (устанавливается через Маркетплейс Bitrix24). Стоимость: от 2 500 ₽/мес.
  2. Кастомная интеграция через API (если нужны нестандартные функции).

Коннектор позволяет:

  • 🔄 Синхронизировать заказы из Bitrix24 в СДЭК и обратно.
  • 📌 Прикреплять трек-номера к сделкам.
  • 📈 Строить отчёты по логистике.

amoCRM

Для amoCRM официального решения нет, но можно использовать:

  • 🤖 Zapier (платный, от 20$/мес).
  • 🛠️ API СДЭК + вебхуки amoCRM (требует разработчика).

MoySklad

MoySklad имеет встроенную интеграцию с СДЭК через приложение «Доставка СДЭК». Настройка занимает 10–15 минут и позволяет:

  • 📦 Автоматически создавать накладные при проведении заказа.
  • 🔄 Подтягивать трек-номера в документы.
  • 📦 Печатать этикетки прямо из MoySklad.

4. Интеграция с маркетплейсами: Ozon, Wildberries, Яндекс Маркет

Если вы продаёте на маркетплейсах, интеграция с СДЭК избавит от ручного ввода заказов. Большинство площадок поддерживают автоматическую передачу заказов в транспортные компании.

Рассмотрим три крупнейших маркетплейса:

Ozon

На Ozon интеграция с СДЭК настраивается в разделе Логистика → Доставка → Подключение транспортных компаний. Для этого:

  1. Выберите СДЭК из списка партнёров.
  2. Введите данные из личного кабинета СДЭК (номер договора, API-ключ).
  3. Настройте правила передачи заказов (например, только для заказов весом до 30 кг).

После настройки заказы будут автоматически уходить в СДЭК со статусом «Готов к отправке».

Wildberries

На Wildberries интеграция работает через сервис WB Партнёр. Алгоритм:

  1. Перейдите в Настройки → Логистика → Транспортные компании.
  2. Выберите СДЭК и заполните реквизиты.
  3. Укажите склады, с которых будет осуществляться отправка.

Важно: Wildberries требует предварительного согласования тарифов с СДЭК. Без этого заказы не будут передаваться.

Яндекс Маркет

На Яндекс Маркете интеграция настраивается через Личный кабинет → Логистика → Транспортные компании. Особенности:

  • 📌 Требуется заключить отдельное соглашение с СДЭК для работы с Яндекс Маркетом.
  • 📦 Поддерживаются только определенные типы грузов (например, не все опасные грузы).
  • 💰 Тарифы могут отличаться от стандартных (уточняйте у менеджера СДЭК).
💡

Если вы продаёте на нескольких маркетплейсах, используйте сервис Мультимаркет или МойСклад для централизованной отправки заказов в СДЭК. Это избавит от необходимости настраивать интеграцию на каждой площадке отдельно.

5. Интеграция с мессенджерами: Telegram, WhatsApp, Viber

Уведомления о статусе доставки в мессенджерах повышают лояльность клиентов. Например, бот в Telegram может отправлять:

  • 📦 Номер заказа и трек-номер после создания накладной.
  • 🚛 Уведомления о изменении статуса (например, «Ваш заказ прибыл в город назначения»).
  • 📌 Ссылку на карту с местоположением курьера (если поддерживается).

Как настроить:

Telegram

Самый простой способ — использовать бота @CDEK_Bot (официальный бот СДЭК). Он позволяет:

  • 🔍 Отслеживать заказы по трек-номеру.
  • 📌 Получать push-уведомления.

Для бизнес-интеграции (например, отправка уведомлений клиентам) потребуется:

  1. Создать своего бота через @BotFather.
  2. Написать скрипт (на Python или PHP), который будет парсить статусы из API СДЭК и отправлять их в чат.
  3. Или использовать сервисы вроде Zapier для связки СДЭК и Telegram.

WhatsApp

Для WhatsApp официального API нет, но можно использовать:

  • 🤖 WhatsApp Business API (платный, от 5 000 ₽/мес).
  • 📲 Сервисы вроде ChatAPI или Twilio.
💡

Интеграция с мессенджерами увеличивает открываемость уведомлений до 80–90% (против 20–30% для email). Это критично для снижения количества невостребованных посылок.

6. Интеграция с 1С: пошаговая инструкция

1С — одна из самых популярных систем для автоматизации бизнеса, и её интеграция с СДЭК позволяет:

  • 📦 Автоматически создавать накладные при проведении заказа.
  • 🔄 Подтягивать трек-номера и статусы обратно в 1С.
  • 📈 Формировать отчёты по логистическим затратам.

Рассмотрим два варианта интеграции:

Вариант 1: Официальный модуль от СДЭК

Стоимость: от 5 000 ₽ (разовая оплата). Подходит для 1С:Управление торговлей и 1С:ERP.

Как настроить:

  1. Скачайте модуль с сайта СДЭК (ссылка).
  2. Установите его в свою конфигурацию 1С.
  3. Введите данные из личного кабинета СДЭК (номер договора, API-ключ).
  4. Настройте правила создания накладных (например, только для оплаченных заказов).

Вариант 2: Самостоятельная интеграция через API

Если вам нужны нестандартные функции (например, работа с нетипичными грузами), можно написать собственную интеграцию.

Пример кода для создания заказа через API СДЭК на 1С:

Функция СоздатьЗаказВСДЭК(Заказ)

Токен ="ваш_api_ключ";


URL ="https://api.cdek.ru/v2/orders";



ТелоЗапроса = Новый Структура;


ТелоЗапроса.Вставить("number", Заказ.Номер);


ТелоЗапроса.Вставить("tariff_code", 136); // Тариф"Дверь-Дверь"


//... другие параметры



Заголовки = Новый Соответствие;


Заголовки.Вставить("Authorization","Bearer" + Токен);


Заголовки.Вставить("Content-Type","application/json");



Ответ = ОтправитьHTTPЗапрос(URL, ТелоЗапроса, Заголовки);


Возврат Ответ;


КонецФункции

Для работы с API потребуется:

  • 🔑 Получить API-ключ в личном кабинете СДЭК (Настройки → Интеграции → API).
  • 📖 Изучить документацию API.
  • 🛠️ Написать обработки для обмена данными (или нанять разработчика).
💡

Перед настройкой интеграции с 1С проверьте, что в вашей конфигурации включена поддержка HTTP-запросов (в некоторых версиях это отключено по умолчанию).

7. Неочевидные интеграции: аналитика, складские системы, IP-телефония

Помимо стандартных интеграций, СДЭК можно связать с менее очевидными сервисами, которые помогут оптимизировать бизнес-процессы.

Аналитика: Google Analytics, Power BI, Яндекс Метрика

Интеграция с системами аналитики позволяет:

  • 📊 Отслеживать конверсию от заказа до доставки.
  • 💰 Анализировать логистические затраты в разрезе регионов или товаров.
  • 📈 Строить дашборды с метриками доставки (например, «Среднее время доставки по городам»).

Как настроить:

  • Использовать Google Tag Manager для отправки событий о статусах доставки.
  • Подключить API СДЭК к Power BI через коннектор.

Складские системы: WMS, «Клеверенс», «Атол»

Если у вас есть складская система (например, WMS), её интеграция с СДЭК позволит:

  • 📦 Автоматически резервировать товары под заказы.
  • 🚛 Оптимизировать маршруты отбора товаров для отправки.
  • 📌 Печатать этикетки и накладные прямо со склада.

IP-телефония: Asterisk, Манго Телефония, Binotel

Интеграция с IP-телефонией полезна для колл-центров. Например, можно настроить:

  • 📞 Автоматический обзвон клиентов при изменении статуса доставки.
  • 🎧 Отображение информации о заказе на экране оператора при входящем звонке.
💡

Интеграция СДЭК с WMS-системой сокращает время сборки заказа на 30–50%, так как исключает ручной ввод данных о грузе.

8. Типичные ошибки при интеграции и как их избежать

Даже при использовании готовых решений возможны ошибки. Вот самые распространённые:

⚠️ Внимание: Если вы используете тестовый API-ключ, заказы не будут реально создаваться в СДЭК. Всегда проверяйте, что в настройках указан производственный ключ перед запуском.
Ошибка Причина Как избежать
Заказы не передаются в СДЭК Неверный API-ключ или номер договора Проверьте данные в личном кабинете СДЭК (Настройки → Интеграции)
Ошибка валидации адреса Неправильный формат адреса (например, без индекса) Используйте сервис проверки адресов СДЭК (ссылка)
Некорректные тарифы Не указаны габариты или вес груза Настройте автоматическое определение веса/габаритов в CRM или 1С
Заказы дублируются Не настроена синхронизация статусов Проверьте настройки вебхуков в личном кабинете СДЭК

Ещё одна частая проблема — превышение лимитов API. Бесплатный тариф СДЭК позволяет делать до 1 000 запросов в день. Если вам нужно больше, обратитесь в поддержку для увеличения лимита.

⚠️ Внимание: При интеграции с маркетплейсами (например, Ozon) проверьте, что в настройках указан правильный склад отправки. Если склад не совпадает с тем, что указан в договоре с СДЭК, заказы не будут передаваться.

FAQ: Частые вопросы об интеграциях СДЭК

Можно ли интегрировать СДЭК с Excel?

Да, но это требует дополнительных инструментов. Варианты:

  • Использовать Power Query для подключения к API СДЭК.
  • Написать макрос на VBA, который будет отправлять запросы в API.
  • Использовать промежуточную систему (например, Google Sheets + Apps Script).

Готового решения «из коробки» нет, но можно автоматизировать выгрузку отчётов из СДЭК в Excel.

Сколько стоит интеграция СДЭК с 1С?

Стоимость зависит от способа интеграции:

  • Официальный модуль от СДЭК: от 5 000 ₽ (разово).
  • Кастомная разработка: от 20 000 ₽ (зависит от сложности).
  • Облачные сервисы (например, «Клеверенс»): от 1 500 ₽/мес.

Если у вас уже есть программист 1С, можно обойтись только стоимостью его работы (от 1 000 ₽/час).

Как проверить, работает ли интеграция?

Создайте тестовый заказ и отследите:

  1. Появилась ли накладная в личном кабинете СДЭК.
  2. Вернулся ли трек-номер в вашу CRM/1С.
  3. Обновился ли статус заказа при изменении статуса в СДЭК.

Если что-то идёт не так, проверьте логи интеграции (обычно они доступны в настройках модуля или в API-ответах).

Можно ли интегрировать СДЭК с Tilda или Shopify?

Да, но способы разные:

  • Tilda: Нет официального плагина, но можно использовать Zapier или кастомный код (JavaScript + API СДЭК).
  • Shopify: Есть официальное приложение CDEK Shipping в Shopify App Store (бесплатно).
Что делать, если API СДЭК не работает?

Проверьте:

  1. Статус работы API на странице статуса СДЭК.
  2. Корректность API-ключа (возможно, он истёк).
  3. Логи запросов (возможно, превышен лимит или неправильный формат данных).
