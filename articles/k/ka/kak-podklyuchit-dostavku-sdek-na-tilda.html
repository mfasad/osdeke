Интеграция СДЭК с Tilda: полное руководство для интернет-магазинов

Подключение доставки СДЭК к сайту на Tilda — ключевой шаг для автоматизации логистики вашего интернет-магазина. Без правильной интеграции вам придётся вручную просчитывать стоимость доставки для каждого заказа, копировать адреса в личный кабинет курьерской службы и отслеживать статус посылок через несколько систем. Это не только отнимает время, но и увеличивает риск ошибок: от неверного тарифа до потери данных о заказе.

В этой статье мы разберём все этапы подключения СДЭК к Tilda — от регистрации в личном кабинете курьерской службы до настройки автоматического расчёта стоимости доставки и трекинга заказов. Особое внимание уделим новым требованиям СДЭК 2026 года к API-интеграциям, которые обязательно учитывать при настройке. Вы узнаете, как избежать типичных ошибок (например, несовпадения веса товара в корзине и в заявке СДЭК), и получите готовые шаблоны для быстрой настройки.

1. Подготовка: что нужно сделать до интеграции

Прежде чем приступать к технической настройке, убедитесь, что выполнены все организационные шаги. Без них интеграция либо не заработает, либо будет работать с ошибками.

Во-первых, зарегистрируйтесь в личном кабинете СДЭК как юридическое лицо или ИП. Физические лица не могут подключать API — это одно из ключевых ограничений службы. Если у вас ещё нет договора с СДЭК, его можно оформить онлайн на сайте cdek.ru (раздел «Для бизнеса»). Процесс занимает от 1 до 3 рабочих дней.

  • 📄 Документы для регистрации: ИНН, ОГРН (или ОГРНИП), выписка из ЕГРЮЛ/ЕГРИП, банковские реквизиты.
  • 🔑 API-ключи: После заключения договора запросите доступ к API в разделе «Интеграции» личного кабинета. Ключи придут на email в течение 24 часов.
  • 📦 Тарифные планы: Определитесь, какие виды доставки будете использовать (курьер, пункты выдачи, посылки до двери) и уточните тарифы для вашего региона.
⚠️ Внимание: Если ваш интернет-магазин работает на Tilda через поддомен (например, vashmagazin.tilda.ws), СДЭК может заблокировать API-ключи из-за политики безопасности. В этом случае используйте собственный домен (например, vashmagazin.ru).

Во-вторых, проверьте данные о товарах в вашем магазине на Tilda. У каждого товара должны быть корректно указаны:

  • 📏 Вес (в граммах или килограммах — СДЭК принимает оба формата).
  • 📦 Габариты (длина × ширина × высота в см). Если габаритов нет, СДЭК будет использовать стандартные значения, что может исказить стоимость доставки.
  • 💰 Стоимость (для расчёта страховки).

2. Настройка личного кабинета СДЭК

После регистрации в СДЭК необходимо настроить параметры, которые будут использоваться при интеграции с Tilda. Этот этап часто упускают, из-за чегоlater возникают ошибки при создании заказов.

Перейдите в раздел Настройки → Профиль компании и проверьте:

  1. Юридические данные: Убедитесь, что ИНН и адрес совпадают с теми, что указаны в договоре.
  2. Контактное лицо: Добавьте email и телефон для уведомлений об ошибках API.
  3. Адреса отправления: Укажите все склады, с которых будет осуществляться отгрузка. Это важно для корректного расчёта тарифов.

Далее перейдите в Интеграции → API и:

  1. Сгенерируйте тестовый и рабочий ключи API (они отличаются!). Тестовый ключ нужен для проверки интеграции, рабочий — для боевого сайта.
  2. Настройте IP-фильтрацию (опционально, но рекомендуется). Укажите IP-адреса серверов Tilda, чтобы защитить ключи от несанкционированного доступа. Список IP Tilda можно найти в документации.
  3. Активируйте опцию «Разрешить создание заказов через API».

3. Подключение СДЭК к Tilda через Zero Block

Tilda не имеет встроенного модуля для интеграции с СДЭК, но это можно сделать с помощью Zero Block (кастомного HTML-кода) или через сервис-посредник вроде API2Cart. Мы рассмотрим первый вариант, так как он бесплатный и не требует сторонних сервисов.

Алгоритм действий:

  1. Добавьте на страницу оформления заказа блок Zero Block (T123).
  2. Вставьте в него следующий код (замените YOUR_API_KEY на ваш ключ СДЭК):
<script src="https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js"></script>

<script>


document.addEventListener('DOMContentLoaded', function() {


// Функция для расчёта стоимости доставки


async function calculateDelivery() {


const cartItems = JSON.parse(localStorage.getItem('tilda-cart'));


const totalWeight = cartItems.reduce((sum, item) => sum + (item.weight || 0), 0);



const response = await axios.post('https://api.cdek.ru/v2/calculator/tariff', {


"type": 1, // 1 - "дверь-дверь", 2 - "дверь-склад", и т.д.


"tariff_code": 136, // Код тарифа (например, 136 - "Курьерская доставка")


"from_location": {


"code": 44 // Код города отправления (Москва)


},


"to_location": {


"code": document.querySelector('[name="city"]').value // Код города получателя


},


"packages": [{


"weight": totalWeight,


"length": 10,


"width": 10,


"height": 10


}]


}, {


headers: {


'Authorization': 'Bearer YOUR_API_KEY',


'Content-Type': 'application/json'


}


});



document.getElementById('delivery-cost').textContent = response.data.price + ' ₽';


}



// Вызываем расчёт при изменении города


document.querySelector('[name="city"]').addEventListener('change', calculateDelivery);


});


</script>

Этот код:

  • 📥 Подгружает данные корзины из localStorage (где Tilda хранит информацию о заказе).
  • ⚖️ Считает общий вес товаров.
  • 🚚 Отправляет запрос в API СДЭК для расчёта стоимости доставки.
  • 💰 Выводит результат на странице (в элемент с ID delivery-cost).

Чтобы код заработал, вам нужно:

  1. Создать на странице элемент для вывода стоимости доставки (например, текстовый блок с ID delivery-cost).
  2. Заменить YOUR_API_KEY на ваш рабочий ключ СДЭК.
  3. Указать правильные коды городов (from_location и to_location). Список кодов можно скачать здесь.

4. Настройка автоматической передачи заказов в СДЭК

Чтобы заказы автоматически создавались в личном кабинете СДЭК, нужно настроить вебхуки (webhooks) в Tilda. Это позволит передавать данные о новых заказах в реальном времени.

Инструкция:

  1. Перейдите в Настройки сайта → Интеграции → Вебхуки.
  2. Добавьте новый вебхук с URL: 
    https://api.cdek.ru/v2/orders

    и выберите событие Новый заказ.

  3. В поле Заголовки добавьте: 
    Authorization: Bearer YOUR_API_KEY

    Content-Type: application/json

  4. В поле Тело запроса вставьте шаблон (пример ниже).

  • {{order_items}} — массив товаров с весом и габаритами.
⚠️ Внимание: Если в вашем заказе несколько товаров с разным весом, нужно динамически рассчитывать общий вес и габариты упаковки. В противном случае СДЭК может неправильно рассчитать стоимость доставки.

- Города доставки (обязательно с кодом СДЭК)

5. Тестирование и отладка интеграции

После настройки обязательно протестируйте интеграцию, чтобы избежать ошибок при реальных заказах. Используйте тестовый ключ API и режимы песчаницы (sandbox) СДЭК.

Порядок тестирования:

  1. Проверка расчёта стоимости:
    • Добавьте товар в корзину.
    • Выберите город доставки.
    • Убедитесь, что стоимость доставки отобразилась корректно.
  2. Тестовый заказ:
    • Создайте заказ с минимальной суммой (например, 1 рубль).
    • Проверьте, что заказ появился в личном кабинете СДЭК (раздел Заказы → В работе).
    • Убедитесь, что данные клиента (ФИО, телефон, адрес) передались без искажений.
  3. Проверка статусов:
    • Измените статус заказа в СДЭК (например, на «В пути»).
    • Убедитесь, что статус обновился в Tilda (если настроена обратная синхронизация).

Типичные ошибки и их решения:

Ошибка Причина Решение
Ошибка 401 (Unauthorized) Неверный API-ключ или истёкший токен Проверьте ключ в настройках. При необходимости сгенерируйте новый в личном кабинете СДЭК.
Ошибка 400 (Bad Request) Некорректные данные в запросе (например, неверный код города) Проверьте формат передаваемых данных. Используйте JSON Lint для валидации JSON.
Стоимость доставки = 0 ₽ Не указан вес товара или неверный тариф Проверьте, что у всех товаров в корзине указан вес. Уточните код тарифа в документации СДЭК.
Заказ не создаётся в СДЭК Не настроен вебхук или ошибка в теле запроса Проверьте логи вебхуков в Tilda (раздел Интеграции → Журнал). Исправьте ошибки в JSON.
Как проверить логи вебхуков в Tilda?

1. Перейдите в Настройки сайта → Интеграции → Вебхуки.

2. Нажмите на название вашего вебхука.

3. Внизу страницы будет блок Журнал выполнения с историей запросов и ответами от СДЭК.

4. Если запрос завершился с ошибкой, нажмите на него, чтобы увидеть детали (код ошибки, тело ответа).

6. Опциональные настройки: трекинг и уведомления

Чтобы клиенты могли отслеживать свои заказы прямо на сайте, а вы получали уведомления об изменениях статуса, настройте дополнительные функции.

1. Отображение трек-номера на сайте:

  • Добавьте на страницу «Спасибо за заказ» блок с текстом:
    • Ваш трек-номер: <span id="track-number"></span>
  • В Zero Block добавьте скрипт, который подтягивает номер заказа из СДЭК: 
    <script>

    // Получаем номер заказа из URL (например, ?order=12345)
    

    const urlParams = new URLSearchParams(window.location.search);
    

    const orderId = urlParams.get('order');
    


    // Запрашиваем данные заказа из СДЭК
    

    fetch(`https://api.cdek.ru/v2/orders/${orderId}`, {
    

    headers: {
    

    'Authorization': 'Bearer YOUR_API_KEY'
    

    }
    

    })
    

    .then(response => response.json())
    

    .then(data => {
    

    document.getElementById('track-number').textContent = data.cdek_number;
    

    });
    

    </script>

2. Уведомления об изменении статуса:

Настройте вебхук в СДЭК, который будет отправлять данные на ваш сайт или email при смене статуса заказа. Для этого:

  1. В личном кабинете СДЭК перейдите в Настройки → Уведомления.
  2. Добавьте новый вебхук с URL вашего обработчика (например, https://vash-site.ru/cdek-status).
  3. Выберите события: Заказ принят на склад, Заказ в пути, Заказ доставлен.

3. Интеграция с Telegram-ботом:

Если вы используете Telegram для уведомлений, можно настроить бота, который будет присылать статус заказов. Пример кода для обработчика:

<?php

$update = json_decode(file_get_contents('php://input'), true);


$trackNumber = $update['cdek_number'];


$status = $update['status'];



// Отправляем уведомление в Telegram


file_get_contents("https://api.telegram.org/botBOT_TOKEN/sendMessage?chat_id=CHAT_ID&text=Заказ%20${trackNumber}%20изменил%20статус%20на:%20${status}");


?>

💡

Настройка уведомлений через вебхуки СДЭК позволяет автоматизировать до 80% рутинных задач по информированию клиентов. Это сокращает количество обращений в поддержку и повышает лояльность покупателей.

7. Альтернативные способы интеграции

Если настройка через Zero Block кажется сложной, можно использовать готовые решения:

1. Сервисы-посредники:

  • 🔗 API2Cart — поддерживает интеграцию Tilda с СДЭК, Boxberry, Почтой России. Стоимость: от 1 500 ₽/месяц.
  • 🔗 RetailCRM — CRM-система с готовым модулем для СДЭК. Подходит для крупных магазинов.
  • 🔗 InSales — если вы готовы мигрировать с Tilda, InSales имеет встроенную интеграцию с СДЭК.

2. Плагины для Tilda:

  • 📦 Tilda Shipping Calculator — плагин для расчёта доставки (включает поддержку СДЭК). Стоимость: 990 ₽ единоразово.
  • 🚀 Tilda Auto Orders — автоматизирует передачу заказов в СДЭК и другие службы. Стоимость: от 2 900 ₽/месяц.

3. Ручное создание заказов:

Если у вас мало заказов (до 10 в день), можно обойтись без автоматизации:

  1. Экспортируйте заказы из Tilda в CSV (раздел Заказы → Экспорт).
  2. Импортируйте их в личный кабинет СДЭК через Заказы → Импорт.
  3. Используйте шаблон СДЭК для корректного формата файла.
⚠️ Внимание: При ручном импорте заказов высока вероятность ошибок в адресах или весе посылок. Всегда проверяйте данные перед отправкой!

8. Оптимизация стоимости доставки для клиентов

Чтобы уменьшить количество отказов на этапе оформления заказа, используйте эти приёмы:

1. Бесплатная доставка от определённой суммы:

  • Настройте в Tilda условие: если сумма заказа > X ₽, скрывайте стоимость доставки или устанавливайте её в 0.
  • Пример кода для Zero Block: 
    if (totalSum >= 5000) {

    document.getElementById('delivery-cost').textContent = '0 ₽ (бесплатно)';
    

    }

2. Скидки на доставку для постоянных клиентов:

  • Используйте купоны в Tilda (раздел Маркетинг → Купоны).
  • Создайте купона типа Скидка на доставку и раздавайте его покупателям после первого заказа.

3. Динамический выбор пункта выдачи:

Добавьте на страницу оформления заказа карту с пунктами выдачи СДЭК. Клиент сможет выбрать удобный ПВЗ прямо на сайте. Пример кода:

<iframe src="https://widget.cdek.ru/widget/?city={city_code}" width="100%" height="500px"></iframe>

Где {city_code} — код города из формы заказа.

4. Предварительный расчёт доставки на странице товара:

Позвольте клиентам увидеть стоимость доставки ещё до добавления товара в корзину. Для этого:

  1. Добавьте на страницу товара поле для ввода города.
  2. Используйте тот же скрипт расчёта, что и на странице оформления заказа.
💡

Если у вас много товаров с одинаковым весом (например, книги или косметика), укажите в настройках СДЭК стандартные габариты упаковки. Это ускорит расчёт доставки и уменьшит количество ошибок.

FAQ: Частые вопросы по интеграции СДЭК с Tilda

🔹 Как узнать код города для API СДЭК?

Список кодов городов можно скачать в документации СДЭК (раздел «Справочники»). Также код города можно получить через API-запрос:

https://api.cdek.ru/v2/location/cities?q=Москва

В ответе будет поле code с нужным значением.

🔹 Можно ли подключить СДЭК к Tilda без знания кода?

Да, для этого используйте готовые сервисы вроде API2Cart или плагины для Tilda (например, Tilda Shipping Calculator). Они предоставляют визуальный интерфейс для настройки интеграции без программирования.

🔹 Почему СДЭК не принимает заказ с весом 0 кг?

СДЭК требует, чтобы у каждого заказа был указан вес (даже минимальный). Если вес не указан, API вернёт ошибку. Решение:

  1. Проверьте, что у всех товаров в Tilda указан вес.
  2. Если вес товара неизвестен, укажите стандартное значение (например, 100 г).
🔹 Как отменить заказ, созданный через API?

Чтобы отменить заказ в СДЭК, отправьте PUT-запрос на:

https://api.cdek.ru/v2/orders/{order_uuid}/cancel

Где {order_uuid} — идентификатор заказа в СДЭК. Пример тела запроса:

{

"reason": "Отмена клиентом"


}
