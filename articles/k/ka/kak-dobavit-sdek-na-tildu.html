Вы создали сайт на Tilda, но клиенты постоянно спрашивают о стоимости доставки СДЭК? Хотите автоматизировать расчёт тарифов и отслеживание посылок прямо на лендинге? Эта статья поможет интегрировать сервисы СДЭК с вашим проектом на Тильде — без программистов и сложных настроек.

В этой инструкции вы найдёте:

  • 🔧 Пошаговое подключение калькулятора СДЭК через HTML-блок
  • 📦 Настройку автоматического расчёта тарифов в корзине Tilda
  • 🔄 Интеграцию с личным кабинетом СДЭК для отслеживания заказов
  • ⚡ Обход ошибок при связке сервисов

Все методы актуальны для 2026 года и работают с последними версиями Tilda Publishing (включая Tilda Zero Block). Если вы используете Tilda Shop, отдельный раздел поможет настроить доставку прямо в админке магазина.

1. Подготовка: что нужно перед интеграцией

Прежде чем добавлять коды и настраивать виджеты, убедитесь, что у вас есть всё необходимое:

  • 📌 Договор с СДЭК (для бизнес-тарифов) или регистрация в личном кабинете для физических лиц. Без этого API-ключи не выдадут.
  • 🔑 API-ключ (если планируете автоматизацию). Его можно получить в разделе Настройки → Интеграции → API.
  • 🖥️ Доступ к админке Tilda с правами редактирования кода (для некоторых методов потребуется Tilda Zero Block).
  • 📦 Список габаритов и веса ваших товаров — это понадобится для точного расчёта тарифов.

Если вы только тестируете интеграцию, используйте песочницу СДЭК (test API). Для этого при регистрации укажите, что вам нужен тестовый доступ. В песочнице все расчёты проводятся с имитацией реальных тарифов, но без оплаты.

⚠️ Внимание: Если вы подключаете СДЭК Бизнес, проверьте, что в договоре указано разрешение на использование API. Без этого пункта ключи для интеграции не выдадут, даже если у вас есть доступ к личному кабинету.

Для упрощения работы подготовьте таблицу с данными ваших посылок:

Тип товара Вес (кг) Габариты (Д×Ш×В, см) Объявленная ценность (₽)
Книга 0.5 20×15×2 1 200
Одежда (пакет) 1.2 30×20×10 3 500
Электроника 2.5 25×25×15 15 000

Эти данные пригодятся при настройке калькулятора, чтобы избежать ошибок в расчётах. Если вы продаёте товары разного веса, лучше создать несколько профилей доставки в СДЭК.

2. Способ 1: Виджет калькулятора СДЭК на странице

Самый простой способ показать клиентам стоимость доставки — встроить готовый виджет СДЭК на сайт. Он позволяет рассчитать тариф прямо на странице товара или в корзине.

Инструкция:

  1. Перейдите в конструктор виджетов СДЭК.
  2. Выберите тип виджета: Калькулятор (для расчёта стоимости) или Отслеживание (для проверки статуса заказа).
  3. Настройте внешний вид: цвет кнопок, шрифты, размеры блока. Для Tilda рекомендуем использовать Тёмную тему или Светлую минималистичную — они лучше вписываются в дизайн шаблонов.
  4. Скопируйте сгенерированный код виджета (начинается с <script src="https://widget.cdek.ru/...>).

Теперь добавьте код на сайт:

  • 🖱️ В Tilda перейдите в раздел Контент → Добавить блок → HTML (T123).
  • 📋 Вставьте скопированный код в поле для HTML.
  • 🔄 Опубликуйте изменения. Виджет появится на странице через 1–2 минуты.

Пример кода для вставки (упрощённый):

<script src="https://widget.cdek.ru/widget.js"></script>

<div


data-type="calc"


data-language="ru"


data-default-city="Москва"


data-hide-destination="false"></div>

⚠️ Внимание: Если виджет не отображается, проверьте, что в настройках Tilda отключён параметр Запретить внешние скрипты (раздел Настройки сайта → Безопасность).

Где лучше размещать виджет:

  • 🛒 На странице корзины — чтобы клиент видел стоимость доставки перед оформлением.
  • 📦 На странице доставки — если у вас отдельный раздел с информацией о логистике.
  • 🏷️ На карточках товаров — если габариты и вес сильно варьируются.

3. Способ 2: Автоматический расчёт в корзине Tilda Shop

Если вы используете Tilda Shop, можно настроить автоматический расчёт доставки СДЭК прямо в корзине. Для этого не нужен API — достаточно подключить готовое решение через Zero Block.

Пошаговая настройка:

  1. В Tilda перейдите в Настройки → Интеграции → Доставка.
  2. Выберите Добавить способ доставки → СДЭК.
  3. Укажите:
    • 📍 Город отправления (должен совпадать с данными в договоре СДЭК).
    • 🔑 API-ключ (если есть) или используйте Без API (расчёт будет менее точным).
    • 📦 Габариты по умолчанию (если не указаны для конкретных товаров).
  • Сохраните настройки и опубликуйте сайт.

    • После настройки клиенты увидят стоимость доставки СДЭК на этапе оформления заказа. Если API не подключён, тариф будет рассчитываться по средним значениям (может отличаться от реальной стоимости на 10–15%).

    Для точной интеграции с API:

    // Пример запроса к API СДЭК для расчёта тарифа

    fetch('https://api.cdek.ru/v2/calculator/tariff', {
    

    method:'POST',
    

    headers: {
    

    'Content-Type':'application/json',
    

    'Authorization':'Token YOUR_API_KEY'
    

    },
    

    body: JSON.stringify({
    

    "tariff_code": 136, // Код тарифа (например, 136 — курьерская доставка)
    

    "from_location": {
    

    "code": 44 // Код города отправления (Москва)
    

    },
    

    "to_location": {
    

    "code": 77 // Код города получения (Санкт-Петербург)
    

    },
    

    "packages": [{
    

    "weight": 1000, // Вес в граммах
    

    "length": 20, // Длина в см
    

    "width": 15, // Ширина в см
    

    "height": 10 // Высота в см
    

    }]
    

    })
    

    })
    

    .then(response => response.json)
    

    .then(data => console.log(data));

    ⚠️ Внимание: При использовании API учитывайте лимиты запросов — в бесплатном тарифе СДЭК разрешает до 1 000 запросов в месяц. Для интернет-магазинов с большим трафиком потребуется бизнес-тариф.
    💡

    Если у вас более 100 заказов в месяц, запросите у СДЭК индивидуальный тариф на API. Это дешевле, чем платить за каждый дополнительный запрос по стандартной цене.

    4. Способ 3: Отслеживание заказов через СДЭК

    Чтобы клиенты могли проверять статус своих посылок прямо на вашем сайте, добавьте виджет отслеживания СДЭК. Он показывает историю движения заказа по ПВЗ или курьеру.

    Как настроить:

    1. Сгенерируйте код виджета отслеживания в конструкторе СДЭК (выберите тип Отслеживание).
    2. В Tilda добавьте блок HTML (T123) на страницу, где должен быть виджет (например, в разделе Мой заказ).
    3. Вставьте код и опубликуйте изменения.

    Пример кода для отслеживания:

    <div

    data-type="track"
    

    data-language="ru"
    

    data-default-track="123456789"></div>

    Где 123456789 — номер трек-номера для примера. Клиенты смогут вводить свои номера прямо в виджете.

    Дополнительные настройки виджета:

    • 🔄 data-hide-input="true" — скрыть поле ввода (если трек-номер передаётся автоматически).
    • 📱 data-mobile="true" — адаптивный дизайн для мобильных устройств.
    • 🎨 data-theme="dark" — тёмная тема оформления.

    Если вы используете Tilda CRM, можно автоматизировать отправку трек-номеров клиентам. Для этого:

    1. В настройках CRM добавьте поле Трек-номер СДЭК.
    2. Настройте триггер, который будет отправлять email с номером отслеживания после создания заказа в СДЭК.
    Как получить трек-номер из СДЭК автоматически?

    Чтобы трек-номера подтягивались в Tilda автоматически, настройте вебхук в личном кабинете СДЭК:

    1. Перейдите в Настройки → Вебхуки.

    2. Укажите URL вашего сайта с обработчиком (например, https://ваш-сайт.tilda.ws/cdek-webhook).

    3. Выберите событие Создание заказа.

    4. В Tilda используйте Zero Block с PHP-кодом для обработки данных.

    Важно: Для этого потребуются навыки программирования или помощь разработчика.

    5. Распространённые ошибки и их решения

    При интеграции СДЭК с Tilda пользователи часто сталкиваются с типичными проблемами. Мы собрали самые частые и способы их исправления.

    Ошибка Причина Решение
    Виджет не отображается Блокировка скриптов в Tilda Отключите Запретить внешние скрипты в настройках безопасности
    Неверный расчёт тарифа Не указаны габариты товара Добавьте вес и размеры в настройках доставки или API-запросе
    Ошибка API:"Invalid token" Неверный API-ключ Проверьте ключ в личном кабинете СДЭК и обновите его в настройках Tilda
    Виджет тормозит сайт Конфликт с другими скриптами Перенесите виджет в отдельный блок или используйте асинхронную загрузку

    Если при расчёте тарифа виджет выдаёт ошибку "Город не найден", проверьте:

    • 🏙️ Правильность написания города (например, "Санкт-Петербург", а не "Питер").
    • 🔍 Код города в API (можно уточнить через справочник СДЭК).
    • 🌍 Регион (некоторые населённые пункты требуют указания области, например, "Московская область, Балашиха").

    Для ускорения загрузки виджета используйте отложенную загрузку (lazy load). Добавьте в код виджета параметр:

    <script src="https://widget.cdek.ru/widget.js" defer></script>
    💡

    Если после всех настроек виджет работает некорректно, проверьте сайт через валидатор W3C. Часто ошибки в вёрстке Tilda ломают работу внешних скриптов.

    6. Продвинутая настройка: API + Tilda Zero Block

    Для полной автоматизации (создание заказов в СДЭК прямо из Tilda, синхронизация статусов) потребуется настройка через API и Zero Block. Этот способ подходит для бизнес-проектов с большим количеством заказов.

    Что можно автоматизировать:

    • 📦 Автоматическое создание заказа в СДЭК после оплаты на сайте.
    • 🔄 Синхронизацию статусов ("В обработке", "Отправлен", "Доставлен").
    • 📊 Генерацию этикеток для печати прямо из админки Tilda.

    Пример кода для создания заказа через API (упрощённая версия):

    // Отправка данных о заказе в СДЭК

    const orderData = {
    

    "type": 1, // Тип заказа (1 — интернет-магазин)
    

    "number":"ORD-" + Math.floor(Math.random * 10000), // Номер заказа
    

    "sender": {
    

    "name":"Ваш магазин",
    

    "phone":"+79991234567"
    

    },
    

    "recipient": {
    

    "name":"Иван Иванов",
    

    "phone":"+79997654321"
    

    },
    

    "from_location": {
    

    "code": 44 // Москва
    

    },
    

    "to_location": {
    

    "code": 77 // Санкт-Петербург
    

    },
    

    "packages": [{
    

    "number":"PKG1",
    

    "weight": 1500,
    

    "length": 20,
    

    "width": 15,
    

    "height": 10
    

    }],
    

    "services": [{
    

    "code":"INSURANCE", // Страховка
    

    "parameter":"10000" // Сумма
    

    }]
    

    };
    


    fetch('https://api.cdek.ru/v2/orders', {
    

    method:'POST',
    

    headers: {
    

    'Content-Type':'application/json',
    

    'Authorization':'Token YOUR_API_KEY'
    

    },
    

    body: JSON.stringify(orderData)
    

    })
    

    .then(response => response.json)
    

    .then(data => {
    

    console.log('Заказ создан в СДЭК:', data);
    

    // Здесь можно сохранить трек-номер в Tilda CRM
    

    });

    Для работы этого кода:

    1. В Tilda добавьте Zero Block на страницу подтверждения заказа.
    2. Вставьте код, заменив YOUR_API_KEY и данные заказа на актуальные.
    3. Настройте обработку ответа от СДЭК (сохранение трек-номера, отправка уведомления клиенту).
    ⚠️ Внимание: При использовании API обязательно настройте обработку ошибок. Например, если СДЭК вернёт ошибку "Превышен лимит запросов", заказ не должен"зависать" в статусе "Оплачен" — клиент должен получить уведомление о проблеме.

    Для тестирования API используйте Postman или cURL. Пример запроса для проверки:

    curl -X POST \

    https://api.cdek.ru/v2/calculator/tariff \
    

    -H'Authorization: Token YOUR_API_KEY' \
    

    -H'Content-Type: application/json' \
    

    -d'{
    

    "tariff_code": 136,
    

    "from_location": {"code": 44},
    

    "to_location": {"code": 77},
    

    "packages": [{"weight": 1000,"length": 20,"width": 15,"height": 10}]
    

    }'

    7. Альтернативные способы интеграции

    Если стандартные методы не подходят, рассмотрите альтернативные варианты:

    • 🔌 Плагины для Tilda: Сервисы вроде Tildalize или TildaTools предлагают готовые модули для СДЭК с расширенными настройками.
    • 🤖 Чат-боты: Настройте бота в Telegram или ВКонтакте, который будет рассчитывать стоимость доставки через API СДЭК и отправлять клиентам.
    • 📧 Email-интеграция: Автоматически отправляйте данные о заказах из Tilda в СДЭК через Zapier или Make (ex-Integromat).

    Пример настройки через Zapier:

    1. Создайте новый Zap с триггером New Order in Tilda.
    2. Добавьте действие Create Order in CDEK.
    3. Сопоставьте поля (имя клиента, адрес, вес посылки и т. д.).
    4. Активируйте интеграцию.

    Стоимость такого решения: от $20/месяц за базовый тариф Zapier. Подходит для магазинов с до 100 заказов в месяц.

    Если вам нужна полная кастомизация (например, уникальный дизайн калькулятора или нестандартная логика расчётов), закажите разработку у Tilda-партнёров. Средняя стоимость такой интеграции — от 15 000 ₽.

    💡

    Перед заказом кастомной интеграции уточните у исполнителя, есть ли у него опыт работы с СДЭК API 2.0. Старая версия API (1.0) больше не поддерживается и может вызвать ошибки.

    8. Оптимизация для SEO и конверсии

    Добавление СДЭК на сайт не только упрощает логистику, но и повышает конверсию и позиции в поиске. Вот как это использовать:

    • 🔍 SEO-тексты: Добавьте на страницу доставки ключевые фразы вроде "доставка СДЭК [ваш город]", "бесплатная доставка СДЭК от [сумма]".
    • ⚡ Микроразметка: Используйте Schema.org для разметки информации о доставке. Пример: 
      <script type="application/ld+json">

      {
      

      "@context":"https://schema.org",
      

      "@type":"OfferShippingDetails",
      

      "shippingRate": {
      

      "@type":"MonetaryAmount",
      

      "value":"350",
      

      "currency":"RUB"
      

      },
      

      "shippingDestination": {
      

      "@type":"DefinedRegion",
      

      "addressCountry":"RU"
      

      },
      

      "deliveryMethod": {
      

      "@type":"ShippingDeliveryMethod",
      

      "name":"Доставка СДЭК"
      

      }
      

      }
      

      </script>

    • 📈 A/B-тестирование: Проверьте, какой вариант размещения калькулятора работает лучше — на странице товара или в корзине.

    Примеры текстов для SEO:

    • 📌 Заголовок H2: "Доставка СДЭК по России: тарифы и сроки для [ваш город]"
    • 📝 Описание: "Узнайте стоимость доставки СДЭК из [ваш город] в любой регион России. Рассчитайте тариф онлайн с помощью нашего калькулятора. Бесплатная доставка при заказе от [сумма] ₽!"

    Чтобы увеличить конверсию:

    • 🎯 Добавьте таймер доставки (например, "Закажите до 15:00 — отправим сегодня!").
    • 💳 Укажите порог бесплатной доставки (например, "Бесплатно при заказе от 3 000 ₽").
    • 📦 Показывайте альтернативные способы доставки (например, самовывоз + СДЭК).

    Используйте тепловую карту (например, Hotjar или Yandex Webvisor), чтобы увидеть, как пользователи взаимодействуют с виджетом СДЭК. Если они не доскролливают до калькулятора, перенесите его выше на странице.

    FAQ: Частые вопросы

    🔹 Можно ли подключить СДЭК к Tilda без API?

    Да, для этого используйте готовый виджет СДЭК (способ 1 в статье). Он работает без API, но расчёт тарифов будет менее точным (например, не учитываются индивидуальные скидки по договору). Для точного расчёта API обязателен.

    🔹 Почему виджет СДЭК не показывает мой город?

    Скорее всего, город неверно указан в настройках виджета или отсутствует в базе СДЭК. Проверьте:

    1. Код города через API СДЭК.
    2. Написание города (должно совпадать с официальным названием в СДЭК).
    3. Регион (например, для "Тюмень" может потребоваться указание "Тюменская область").

    Если город новый или небольшой, свяжитесь с поддержкой СДЭК для добавления его в базу.

    🔹 Как сделать, чтобы стоимость доставки автоматически добавлялась в заказ?

    Для этого:

    1. В Tilda Shop настройте способ доставки СДЭК (раздел Настройки → Доставка).
    2. Укажите API-ключ (если есть) или используйте ручной ввод тарифов.
    3. В настройках корзины включите опцию "Показывать стоимость доставки".

    Если API не подключён, клиент увидит приблизительную стоимость, а точную сумму вы сможете уточнить при обработке заказа.

    🔹 Сколько стоит интеграция СДЭК с Tilda?

    Стоимость зависит от способа интеграции:

    • Бесплатно: Виджет калькулятора или отслеживания (без API).
    • От 0 ₽/месяц: API СДЭК (бесплатно до 1 000 запросов, затем от 0.5 ₽ за запрос).
    • От 15 000 ₽: Кастомная интеграция через Zero Block (разовая оплата разработчику).
    • От $20/месяц: Автоматизация через Zapier или аналоги.
    🔹 Можно ли отслеживать заказы СДЭК прямо в Tilda CRM?

    Да, для этого:

    1. Настройте вебхук в личном кабинете СДЭК (раздел Настройки → Вебхуки).
    2. Укажите URL вашего сайта с обработчиком (например, https://ваш-сайт.tilda.ws/cdek-webhook).
    3. В Tilda используйте Zero Block с PHP-кодом для обработки данных и обновления статусов в CRM.

    Для этого потребуются базовые навыки программирования или помощь разработчика.
