Подключение СДЭК к Tilda: полная инструкция с примерами кода

Интеграция услуг СДЭК с платформой Tilda позволяет автоматизировать расчёт стоимости доставки, отображать пункты выдачи на карте и упростить обработку заказов для интернет-магазинов. Однако многие сталкиваются с техническими нюансами: от получения API-ключей до корректной настройки JavaScript-виджета в конструкторе. В этой статье разберём все этапы — от регистрации в личном кабинете СДЭК до тестирования работы калькулятора на живом сайте.

Особенность интеграции заключается в том, что Tilda не поддерживает прямую установку плагинов, поэтому подключение осуществляется через вставку пользовательского HTML-кода. Это требует базовых знаний работы с API и структурой данных СДЭК. Мы рассмотрим два основных сценария: добавление калькулятора стоимости доставки и внедрение виджета выбора пунктов выдачи — с учётом актуальных тарифов 2026 года и требований к оформлению заказов.

1. Подготовка: регистрация в личном кабинете СДЭК

Прежде чем подключать сервисы СДЭК к Tilda, необходимо получить доступ к API компании. Для этого:

  • 📝 Зарегистрируйтесь на официальном сайте СДЭК как юридическое лицо или ИП. Физические лица не могут подключать API.
  • 🔑 После подтверждения аккаунта перейдите в раздел Настройки → Интеграции → API и сгенерируйте ключ доступа (он потребуется для всех запросов).
  • 📋 Убедитесь, что в личном кабинете заполнены все реквизиты компании, включая адрес склада (это влияет на расчёт тарифов).
  • 📦 Активируйте услугу «Интернет-магазин» в разделе Тарифы и услуги — без неё API-калькулятор не будет работать.

Важно: ключ API — конфиденциальная информация. Никогда не публикуйте его в открытом доступе (например, в репозиториях GitHub или в коде страницы без шифрования). Для Tilda это неактуально, но при разработке тестовых страниц на других платформах используйте переменные окружения.

⚠️ Внимание: Если ваш аккаунт в СДЭК зарегистрирован менее 3 дней назад, API может быть временно заблокирован для новых пользователей. В этом случае обратитесь в поддержку с запросом на разблокировку, указав цель интеграции (например, «Подключение калькулятора к сайту на Tilda»).

2. Выбор способа интеграции: калькулятор vs виджет пунктов выдачи

СДЭК предлагает два основных инструмента для сайтов:

  1. Калькулятор стоимости доставки — рассчитывает цену в зависимости от веса, габаритов и адреса получателя. Подходит для страниц корзины или оформления заказа.
  2. Виджет пунктов выдачи — интерактивная карта с адресами ПВЗ, где пользователь может выбрать удобный пункт самовывоза. Оптимален для страниц доставки.

Выбор зависит от логики вашего магазина:

Критерий Калькулятор стоимости Виджет ПВЗ
Цель Показать цену доставки до оформления заказа Предложить выбор пункта самовывоза
Где размещать Страница корзины, блок «Доставка» Страница «Контакты» или «Доставка»
Техническая сложность Средняя (требуется настройка параметров) Низкая (готовый код от СДЭК)
Дополнительные данные Вес, габариты товара, адрес получателя Город получателя (опционально)

Многие интернет-магазины используют оба инструмента одновременно. Например, на странице корзины размещают калькулятор, а на странице «Доставка» — виджет ПВЗ. Это уменьшает количество вопросов от клиентов и сокращает время оформления заказа.

3. Подключение калькулятора стоимости доставки

Для вставки калькулятора в Tilda выполните следующие шаги:

  1. Получите код калькулятора в личном кабинете СДЭК: 
    <script src="https://api.cdek.ru/v2/calculator/widget.js"></script>

    <div
    

    data-api-key="ВАШ_API_КЛЮЧ"
    

    data-default-weight="500"
    

    data-default-length="10"
    

    data-default-width="10"
    

    data-default-height="10"></div>

    Замените ВАШ_API_КЛЮЧ на реальный ключ из личного кабинета. Параметры data-default-* укажите согласно средним габаритам ваших посылок (в сантиметрах и граммах).

  2. Добавьте код в Tilda:
    • 🖥️ Откройте редактор страницы, куда нужно вставить калькулятор.
    • 📌 Добавьте блок HTML (раздел Другие → HTML).
    • 🔧 Вставьте скопированный код в поле блока.
  • Настройте стили (опционально):

    Калькулятор наследует стили вашего сайта, но при необходимости можно добавить CSS-правила в блок T123->Settings->Custom CSS. Например, чтобы изменить цвет кнопки:

    .cdek-calculator button {

    background-color: #ff6b00 !important;
    

    }

    • После сохранения изменений калькулятор появится на странице. Проверьте его работу, указав разные города и параметры посылки. Если калькулятор не отображается, убедитесь, что:

    • 🔄 Код вставлен в правильный блок (не в Текст или Заголовок).
    • 🔑 API-ключ корректный и не содержит лишних символов.
    • 🌍 В настройках СДЭК указан верный регион отправления (склад).

    Код вставлен в блок HTML|API-ключ заменён на реальный|Параметры веса/габаритов указаны|Страница опубликована (не в режиме редактирования)|Проверено в разных браузерах-->

    ⚠️ Внимание: Если ваш сайт на Tilda использует протокол HTTP (без шифрования), калькулятор СДЭК не будет работать по соображениям безопасности. Обязательно подключите HTTPS в настройках домена.

    4. Интеграция виджета пунктов выдачи (ПВЗ)

    Виджет ПВЗ позволяет клиентам выбрать удобный пункт самовывоза прямо на сайте. Его подключение проще, чем калькулятора, так как не требует указания габаритов посылки.

    Инструкция:

    1. Сгенерируйте код виджета в личном кабинете СДЭК: 
      <script src="https://api.cdek.ru/v2/pvz/widget.js"></script>

      <div
      

      data-api-key="ВАШ_API_КЛЮЧ"
      

      data-city="Москва"></div>

      Параметр data-city можно оставить пустым — тогда виджет будет определять город автоматически по IP пользователя, либо указать город по умолчанию (например, data-city="Санкт-Петербург").

    2. Вставьте код в Tilda аналогично калькулятору (через блок HTML).
    3. Настройте отображение:

      Виджет поддерживает два режима:

      • 🗺️ Карта — интерактивная карта с метками ПВЗ (режим по умолчанию).
      • 📋 Список — текстовый перечень пунктов с адресами. Чтобы активировать, добавьте параметр data-view="list".

    Пример кода для отображения списка ПВЗ:

    <div

    data-api-key="ВАШ_API_КЛЮЧ"
    

    data-city="Екатеринбург"
    

    data-view="list"></div>

    Если виджет не показывает пункты выдачи, проверьте:

    • 🏙️ Город указан корректно (например, «Нижний Новгород», а не «Нижний»).
    • 🔍 В личном кабинете СДЭК активирована опция «Пункты выдачи» для вашего тарифа.
    • 📡 На сайте нет блокировщиков скриптов (например, AdBlock может блокировать загрузку виджета).
    💡

    Если виджет ПВЗ тормозит загрузку страницы, добавьте параметр data-lazy="true" — это отложит загрузку карты до момента прокрутки пользователем до блока.

    5. Настройка дополнительных параметров

    Оба виджета (калькулятор и ПВЗ) поддерживают расширенные настройки через data-атрибуты. Вот наиболее полезные из них:

    Атрибут Описание Пример значения Для какого виджета
    data-default-weight Вес посылки по умолчанию (граммы) 1000 Калькулятор
    data-hide-delivery Скрыть вариант «Доставка курьером» true Калькулятор
    data-limit Ограничить количество показываемых ПВЗ 10 Виджет ПВЗ
    data-show-all Показать все ПВЗ, включая временно закрытые false Виджет ПВЗ
    data-language Язык интерфейса (ru/en) en Оба

    Пример калькулятора с скрытой курьерской доставкой и фиксированным весом 2 кг:

    <div

    data-api-key="ВАШ_API_КЛЮЧ"
    

    data-default-weight="2000"
    

    data-hide-delivery="true"></div>

    Важно: если вы используете мультирегиональность на сайте (например, продаёте в несколько стран), укажите параметр data-country для корректного отображения тарифов. Для России значение — RU, для Казахстана — KZ.

    6. Тестирование и устранение ошибок

    После настройки виджетов обязательно протестируйте их работу в разных сценариях:

    • 📦 Корректность расчётов: сравните стоимость доставки в калькуляторе на сайте и в личном кабинете СДЭК для одинаковых параметров.
    • 🌍 Геолокация: проверьте, верно ли виджет ПВЗ определяет город пользователя (используйте VPN для эмуляции разных регионов).
    • 📱 Адаптивность: убедитесь, что виджеты корректно отображаются на мобильных устройствах (особенно карта ПВЗ).
    • 🔄 Кэширование: после изменений в коде очистите кэш браузера (Ctrl+F5), так как Tilda может сохранять старые версии скриптов.

    Типичные ошибки и их решения:

    Ошибка Возможная причина Решение
    Виджет не отображается Некорректный API-ключ или блокировка скриптов Проверьте ключ и отключите адблоки в браузере
    Калькулятор показывает «Нет тарифов» Не указан вес/габариты или неправильный город Добавьте параметры data-default-* и проверьте город
    Карта ПВЗ не загружается Конфликт с другими скриптами на странице Перенесите виджет в отдельный блок или отключите другие скрипты
    Цены в калькуляторе не совпадают с ЛК СДЭК Указан неверный склад отправления Проверьте адрес склада в настройках СДЭК

    Если ошибка не устраняется, воспользуйтесь инструментом разработчика в браузере (F12 → вкладка Console). Ошибки API СДЭК обычно содержат код (например, 403 Forbidden — неверный ключ, 400 Bad Request — некорректные параметры).

    Пример ошибки в консоли браузера

    {

    "request_id": "123456789",

    "errors": [

    {

    "code": "invalid_api_key",

    "message": "Неверный API-ключ"

    }

    ]

    }

    7. Автоматизация обработки заказов

    Интеграция виджетов СДЭК с Tilda решает только часть задач — расчёт стоимости и выбор ПВЗ. Для полной автоматизации обработки заказов потребуется:

    1. Настройка вебхуков:

      В личном кабинете СДЭК перейдите в Настройки → Вебхуки и укажите URL вашего сайта, куда будут отправляться уведомления о статусах заказов. Для Tilda это возможно только при использовании внешнего бэкенда (например, Google Sheets + Zapier или собственный сервер).

    2. Экспорт заказов в СДЭК:

      После оформления заказа на сайте его данные нужно передать в СДЭК для формирования накладной. Это можно сделать:

      • 📤 Вручную — копировать данные из Tilda в ЛК СДЭК.
      • 🤖 Автоматически — через API (требуется разработчик или сервис-интегратор вроде Albato).
  • Отслеживание заказов:

    Чтобы клиенты могли отслеживать посылки на вашем сайте, добавьте виджет трекинга СДЭК:

    <script src="https://api.cdek.ru/v2/tracking/widget.js"></script>

    <div data-api-key="ВАШ_API_КЛЮЧ"></div>

    • Для небольших магазинов достаточно ручного экспорта заказов. Если заказов много (от 50 в день), рекомендуется автоматизировать процесс через API. Например, можно настроить интеграцию Tilda → Google Таблицы → СДЭК с помощью Zapier или Make (ex-Integromat).

    💡

    Без автоматизации обработки заказов виджеты СДЭК на сайте теряют половину своей эффективности. Клиенты увидят стоимость доставки, но вам всё равно придётся вручную создавать закзазы в ЛК СДЭК.

    8. Оптимизация для SEO и конверсии

    Виджеты СДЭК не только упрощают логистику, но и влияют на поведенческие факторы сайта. Чтобы максимально использовать их потенциал:

    • 📈 Добавьте микроразметку для пунктов выдачи. Это поможет поисковым системам понимать, что на странице есть адреса ПВЗ. Пример: 
      <script type="application/ld+json">

      {
      

      "@context": "https://schema.org",
      

      "@type": "Organization",
      

      "name": "Пункты выдачи СДЭК",
      

      "url": "https://ваш-сайт.ru/delivery",
      

      "address": {
      

      "@type": "PostalAddress",
      

      "addressCountry": "RU",
      

      "addressLocality": "Москва",
      

      "streetAddress": "ул. Ленина, д. 1"
      

      }
      

      }
      

      </script>

    • 🔍 Оптимизируйте страницу доставки под ключевые запросы вроде «доставка [ваш город] СДЭК» или «пункты выдачи [ваш город]». Включите эти фразы в заголовки и описания.
    • 📊 Отслеживайте конверсию с помощью Google Analytics или Яндекс.Метрики. Настройте цели для событий «Расчёт доставки» и «Выбор ПВЗ».
    • 💬 Добавьте FAQ по доставке на страницу с виджетами. Это снизит количество обращений в поддержку. Пример вопросов:
      • «Сколько стоит доставка в [город]?»
      • «Как изменить пункт выдачи после оформления заказа?»

    Также рекомендуется добавить на страницу доставки блок с преимуществами работы с СДЭК (например, «Доставка за 1–2 дня», «Более 3000 пунктов выдачи», «Страхование груза»). Это увеличит доверие клиентов и сократит количество отказов на этапе оформления заказа.

    💡

    Используйте A/B-тестирование для сравнения конверсии страниц с виджетом СДЭК и без него. Часто добавление калькулятора увеличивает количество оформленных заказов на 15–20%.

    FAQ: Частые вопросы по интеграции СДЭК и Tilda

    Можно ли подключить СДЭК к Tilda без API-ключа?

    Нет, для работы виджетов обязательно нужен API-ключ из личного кабинета СДЭК. Без него скрипты не смогут получать данные о тарифах и пунктах выдачи. Альтернатива — использовать готовые решения вроде сервиса Boxberry, но они менее гибкие.

    Почему калькулятор показывает «Нет доступных тарифов»?

    Эта ошибка возникает в нескольких случаях:

    • В личном кабинете СДЭК не активирована услуга «Интернет-магазин».
    • Указан неверный город или регион отправления (проверьте адрес склада).
    • Вес или габариты посылки превышают лимиты тарифа (например, для «Посылки» максимальный вес — 30 кг).
    • API-ключ заблокирован за превышение лимита запросов (бесплатный тариф позволяет 1000 запросов в день).

    Проверьте каждый пункт и при необходимости обратитесь в поддержку СДЭК.

    Как изменить дизайн виджета под стиль моего сайта?

    Виджеты СДЭК поддерживают кастомизацию через CSS. Основные классы для стилизации:

    • .cdek-calculator — общий контейнер калькулятора.
    • .cdek-pvz-map — блок с картой ПВЗ.
    • .cdek-pvz-list — список пунктов выдачи.
    • .cdek-button — кнопки внутри виджетов.

    Пример кода для изменения цвета кнопки:

    .cdek-button {

    background-color: #ff6b00 !important;
    

    border-radius: 20px !important;
    

    }

    Добавьте этот код в T123 → Settings → Custom CSS.

    Можно ли подключить СДЭК к Tilda через Zero Block?

    Технически да, но это требует знаний JavaScript и работы с API Tilda. Zero Block позволяет создавать динамические элементы, но для интеграции с СДЭК придётся самостоятельно писать код для отправки запросов к API доставки и обработки ответов. Готовых решений для такой интеграции нет — проще использовать стандартные виджеты, описанные в статье.

    Сколько стоит подключение СДЭК к сайту?

    Сама интеграция виджетов на сайт Tilda бесплатна. Однако учтите следующие расходы:

    • API СДЭК: бесплатно до 1000 запросов в день. Для большего трафика — от 500 руб./мес.
    • Доставка: тарифы зависят от веса, габаритов и направления (от 150 руб. за посылку).
    • Автоматизация: если нужна интеграция с CRM или 1С, потребуется разработчик (от 10 000 руб.).

    Точную стоимость доставки для вашего региона можно уточнить в калькуляторе СДЭК.
