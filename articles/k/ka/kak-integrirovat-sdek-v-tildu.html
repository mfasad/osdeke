Почему интеграция СДЭК с Tilda выгодна для бизнеса

СДЭК остаётся лидером среди курьерских служб в России по объёму обработки заказов e-commerce, а Tilda — одна из самых популярных платформ для создания лендингов и интернет-магазинов. Их совместное использование позволяет автоматизировать до 70% рутинных операций: от расчёта стоимости доставки до генерации этикеток для посылок. Главное преимущество такой интеграции — снижение количества ошибок при оформлении заказов и ускорение обработки до 2-3 минут на одну посылку.

Без интеграции менеджерам приходится вручную перепечатывать адреса из заказов Тильды в личный кабинет СДЭК, сверять вес и габариты, а затем отправлять клиенту номер трека. Автоматизация убирает эти этапы: данные передаются напрямую, а покупатель получает трек-номер сразу после оплаты. По данным СДЭК, такие магазины сокращают количество возвратов на 12-15% за счёт точного расчёта стоимости доставки ещё на этапе оформления.

В этой статье разберём все способы интеграции — от простого встраивания виджетов до полноценной синхронизации через API. Вы узнаете, как:

  • 📦 Подключить калькулятор доставки СДЭК на страницу оформления заказа
  • 🔄 Настроить автоматическую передачу заказов из Тильды в личный кабинет СДЭК
  • 📊 Внедрить отслеживание посылок прямо на сайте
  • 💳 Синхронизировать тарифы и пункты выдачи с актуальными данными

Шаг 1: Установка виджета калькулятора СДЭК на сайт Tilda

Виджет позволяет покупателям рассчитать стоимость доставки прямо на странице оформления заказа. Для его установки:

  1. Перейдите в личный кабинет СДЭК в раздел Виджеты → Калькулятор доставки.
  2. Настройте параметры:
    • 📌 Тип тарифа: выберите "Интернет-магазин" или "Курьер"
    • 📦 Габариты по умолчанию: укажите средний вес и размеры ваших посылок
    • 🏠 Город отправления: добавьте ваш склад (например, Москва)
  • Скопируйте сгенерированный код виджета.

    • Теперь добавьте виджет в Tilda:

    1. Откройте страницу оформления заказа в редакторе Tilda.
    2. Добавьте блок HTML (Т123 → Дополнительно → HTML-код).
    3. Вставьте скопированный код и сохраните изменения.

    Важный нюанс: если ваш магазин работает с предоплатой, настройте в виджете опцию "Расчёт с учётом предоплаты". Это снизит стоимость доставки для покупателей на 10-30% в зависимости от тарифа.

    ⚠️ Внимание: Виджет СДЭК может конфликтовать с другими скриптами на странице (например, с аналитикой или чат-ботами). Перед публикацией проверьте консоль браузера (F12 → Console) на наличие ошибок типа "Uncaught ReferenceError".

    Шаг 2: Настройка передачи заказов из Tilda в СДЭК

    Чтобы заказы автоматически попадали в личный кабинет СДЭК, нужно настроить интеграцию через JS-код или API. Рассмотрим оба варианта.

    Опция 1: JS-код для формы заказа (без API)

    Этот метод подходит, если у вас нет доступа к серверу или вы не хотите использовать API. Алгоритм:

    1. В Tilda перейдите в настройки формы заказа (Настройки → Формы → Ваша форма).
    2. В разделе "Действия после отправки" выберите "JavaScript".
    3. Вставьте следующий код (замените YOUR_ACCOUNT и YOUR_SECURE_PASSWORD на данные из личного кабинета СДЭК):
    // Функция для отправки заказа в СДЭК

    function sendToCdek(orderData) {
    

    const cdekAuth = btoa('YOUR_ACCOUNT:YOUR_SECURE_PASSWORD');
    

    const url = 'https://api.cdek.ru/v2/orders';
    


    fetch(url, {
    

    method: 'POST',
    

    headers: {
    

    'Authorization': `Basic ${cdekAuth}`,
    

    'Content-Type': 'application/json'
    

    },
    

    body: JSON.stringify({
    

    type: 1, // Тип заказа: 1 - интернет-магазин
    

    number: orderData.order_id,
    

    tariff_code: 137, // Код тарифа (например, 137 - курьерская доставка)
    

    from_location: {
    

    code: 44, // Код города отправления (Москва)
    

    },
    

    to_location: {
    

    code: orderData.city_code, // Код города получателя
    

    },
    

    packages: [{
    

    weight: orderData.weight,
    

    length: orderData.length,
    

    width: orderData.width,
    

    height: orderData.height
    

    }]
    

    })
    

    })
    

    .then(response => response.json())
    

    .then(data => console.log('Заказ передан в СДЭК:', data))
    

    .catch(error => console.error('Ошибка:', error));
    

    }
    


    // Получение данных из формы Tilda
    

    window.addEventListener('tildaFormSubmitSuccess', function(e) {
    

    const formData = e.detail;
    

    sendToCdek(formData);
    

    });

    Этот код отправляет в СДЭК только базовые данные. Для полной синхронизации (включая ФИО, адрес, телефон) нужно расширить объект orderData.

    Опция 2: Полноценная интеграция через API СДЭК

    Для крупных магазинов рекомендуем использовать REST API СДЭК. Преимущества:

    • 🔄 Двусторонняя синхронизация (заказы + статусы доставки)
    • 📦 Автоматическое создание этикеток
    • 💰 Актуальные тарифы без ручного обновления

    Для настройки:

    1. Зарегистрируйте приложение в личном кабинете разработчика СДЭК.
    2. Получите client_id и client_secret.
    3. Используйте Tilda API + СДЭК API для обмена данными. Пример схемы: 
      Tilda (заказ) → Ваш сервер (обработка) → СДЭК API (создание заказа) → СДЭК (отправка)
    ⚠️ Внимание: При работе с API СДЭК обязательно настройте webhook для получения статусов доставки. Без этого покупатели не смогут отслеживать заказы на вашем сайте. Инструкция по настройке: документация СДЭК.
    💡

    Если вы не разбираетесь в API, используйте готовые сервисы-коннекторы, такие как Shopolog или Alfa-Bank Эквайринг. Они предлагают плагины для Tilda с предварительно настроенной интеграцией СДЭК.

    Шаг 3: Добавление отслеживания заказов на сайт

    Чтобы покупатели могли проверять статус доставки не покидая ваш сайт, интегрируйте виджет отслеживания. Для этого:

    1. В личном кабинете СДЭК перейдите в Виджеты → Отслеживание заказов.
    2. Настройте дизайн виджета (цвета, шрифты) под стиль вашего сайта.
    3. Скопируйте код и вставьте его в блок HTML на странице "Мои заказы" или в футере сайта.

    Пример кода для вставки:

    <div

    data-account="YOUR_ACCOUNT"
    

    data-style="light"
    

    data-default-language="ru"></div>
    

    <script src="https://widget.cdek.ru/widget/tracking.js"></script>

    Как кастомизировать виджет отслеживания?

    Виджет поддерживает следующие параметры кастомизации через атрибуты data-*:

    - data-style="light/dark" — светлая или тёмная тема

    - data-default-language="ru/en" — язык интерфейса

    - data-show-map="true/false" — отображение карты маршрута

    - data-compact="true" — компактный вид для мобильных устройств

    Пример:

    <div data-account="12345" data-style="dark" data-show-map="true"></div>

    Шаг 4: Синхронизация пунктов выдачи и тарифов

    Чтобы покупатели видели актуальные пункты выдачи и тарифы, настройте их автоматическое обновление. Для этого:

    1. Используйте API СДЭК для получения списка ПВЗ: 
      GET https://api.cdek.ru/v2/deliverypoints?city_code=44

      (где 44 — код города, например, Москва).

    2. Обработайте ответ и динамически загрузите пункты в форму заказа Tilda.

    Пример JS-кода для загрузки ПВЗ:

    // Получение пунктов выдачи для Москвы (код города 44)

    fetch('https://api.cdek.ru/v2/deliverypoints?city_code=44', {
    

    headers: {
    

    'Authorization': 'Bearer YOUR_ACCESS_TOKEN'
    

    }
    

    })
    

    .then(response => response.json())
    

    .then(data => {
    

    const select = document.getElementById('delivery-points');
    

    data.forEach(point => {
    

    const option = document.createElement('option');
    

    option.value = point.code;
    

    option.textContent = `${point.name} (${point.address})`;
    

    select.appendChild(option);
    

    });
    

    });

    Для обновления тарифов используйте эндпоинт:

    POST https://api.cdek.ru/v2/calculator/tarifflist

    Отправляйте в теле запроса данные о весе, габаритах и городе доставки.

    ⚠️ Внимание: Тарифы СДЭК обновляются ежедневно в 03:00 МСК. Если вы кэшируете данные на своём сервере, настройте автоматическое обновление кэша в это время, иначе покупатели увидят устаревшие цены.

    Шаг 5: Автоматизация печати этикеток и документов

    После передачи заказа в СДЭК можно автоматически генерировать этикетки и товарные накладные. Для этого:

    1. В личном кабинете СДЭК включите опцию "Автопечать документов" в настройках интеграции.
    2. Настройте шаблон этикетки (размер 100×100 мм или 100×70 мм).
    3. Используйте API СДЭК для получения PDF-версии этикетки: 
      GET https://api.cdek.ru/v2/orders/{order_uuid}/print/label

    Пример кода для скачивания этикетки:

    // Получение этикетки для заказа с UUID

    fetch(`https://api.cdek.ru/v2/orders/${orderUuid}/print/label`, {
    

    headers: {
    

    'Authorization': 'Bearer YOUR_ACCESS_TOKEN'
    

    }
    

    })
    

    .then(response => response.blob())
    

    .then(blob => {
    

    const url = window.URL.createObjectURL(blob);
    

    const a = document.createElement('a');
    

    a.href = url;
    

    a.download = `label_${orderUuid}.pdf`;
    

    a.click();
    

    });

    Для печати на термопринтере используйте библиотеку jsPDF или специализированные драйверы (например, Zebra Browser Print).

    💡

    Автоматическая печать этикеток сокращает время обработки заказа на 40-60% и исключает ошибки при ручном заполнении документов.

    Распространённые ошибки и их решения

    При интеграции СДЭК с Tilda часто возникают следующие проблемы:

    Ошибка Причина Решение
    Виджет не отображается Конфликт с другими скриптами Перенесите код виджета в начало страницы или используйте setTimeout
    Неверный расчёт стоимости Не указаны габариты по умолчанию Настройте средние вес и размеры в личном кабинете СДЭК
    Заказы не передаются в СДЭК Неверные данные авторизации Проверьте client_id и client_secret в коде
    Ошибка 403 при запросе к API Истёк токен доступа Обновите токен через /v2/oauth/token

    Если после настройки интеграции заказы дублируются в СДЭК, проверьте:

    • 🔍 Наличие уникального идентификатора заказа (order_id) в запросе
    • 🔄 Логику повторной отправки данных (например, при обновлении страницы)
    • 📝 Настройки webhook в личном кабинете СДЭК

    Критическая ошибка: если в ответ на запрос к API СДЭК вы получаете статус 429 Too Many Requests, это означает превышение лимита запросов (1000 запросов/час для стандартного тарифа). Решение: оптимизируйте код, добавив кэширование ответов или перейдите на расширенный тариф.

    FAQ: Частые вопросы по интеграции СДЭК и Tilda

    Можно ли интегрировать СДЭК с Tilda без программиста?

    Да, для этого используйте готовые сервисы-коннекторы:

    • Shopolog — поддерживает автоматическую передачу заказов и отслеживание
    • Alfa-Bank Эквайринг — если у вас подключён эквайринг Альфа-Банка
    • RetailCRM — для комплексного управления заказами

    Эти сервисы предлагают плагины для Tilda с предварительно настроенной интеграцией.

    Как обновить тарифы СДЭК на сайте, если они изменились?

    Тарифы обновляются автоматически при каждом запросе к API СДЭК. Если вы используете виджет калькулятора, данные подтягиваются в реальном времени. Для кэшированных данных (например, в базе вашего сайта) настройте cron-задачу на ежедневное обновление через API-метод /v2/calculator/tarifflist.

    Почему покупатели не видят пункты выдачи в своей локации?

    Проблема может быть в следующем:

    • Не указан код города получателя в запросе к API
    • Фильтр по типу ПВЗ (например, только "постаматы") исключает другие варианты
    • В личном кабинете СДЭК отключена опция "Показывать все ПВЗ"

    Проверьте параметры запроса и настройки аккаунта.

    Можно ли интегрировать СДЭК с Tilda Zero Block?

    Да, но с ограничениями. В Tilda Zero Block нельзя добавлять произвольный HTML-код, поэтому:

    • Виджет калькулятора вставить не получится
    • Можно использовать только JS-код в настройках формы или внешние сервисы (например, Shopolog)
    • Для полноценной интеграции потребуется перейти на платный тариф Tilda

    Как тестировать интеграцию перед запуском?

    СДЭК предоставляет песочницу (sandbox) для тестирования API. Чтобы её подключить:

    1. Зарегистрируйте тестовый аккаунт на https://api.edu.cdek.ru/
    2. Используйте тестовые данные (например, город с кодом 137 — Санкт-Петербург)
    3. Проверьте корректность передачи заказов и генерации этикеток

    В песочнице лимиты запросов выше (до 5000/час), а тарифы фиксированные.
